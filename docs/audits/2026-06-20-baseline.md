## 3. 已知问题清单 + 优化建议

按性价比从高到低排序，建议在 P1 / P2 polish 阶段消化。

### 3.1 Performance（首页 70 / CH1 67）— 优先级：高

共同根因：未做 production 级别的资源优化（无 fingerprint、无 minify、无 deferred module 拆分）。

1. **JS / CSS minify & 内容哈希**
   - 首页：CSS 节省 12 KiB，JS 节省 8 KiB。
   - CH1 slide 1：CSS 节省 20 KiB，JS 节省 60 KiB（slide 引擎 + 动画模块为主）。
   - 建议：保留 dev 源文件，CI 增加 minify 任务，运行时只引 `.min.css` / `.min.js`。
2. **未使用 CSS 清理（unused-css-rules）**
   - 首页 13 KiB，CH1 63 KiB。
   - 建议：检查并裁剪 `slides.css` / `main.css` / `themes.css` 中未触达的选择器；
     CH1 路径因 `slide-engine.js` 启动时按需注入，CSS 中可能存在未在静态首屏触达的样式。
3. **LCP / FCP 偏慢（3.5–6.0 s）**
   - 根因：headless 模式 + dev server + 多模块同步 `<script type="module">` 串行加载。
   - 建议：给关键 CSS 加 `<link rel="preload" as="style">`，主入口脚本加 `defer`；考虑在 production 把
     `js/main.js` / `js/slides-main.js` 拆分为 critical + lazy 两段。
4. **Total byte weight 590 / 752 KiB**
   - 建议：图片资源（`assets/images/*.jpg`）未做 WebP/AVIF 转换，hero / cover 图应增加响应式 `srcset`。

### 3.2 Accessibility（首页 95 / CH1 94）— 优先级：中

多数为中-低严重度，可在 polish 阶段一并修复。

1. **`aria-allowed-role`（home 65、ch1 1）**
   - 大量装饰性 SVG 上的 `role` 与元素不匹配，建议对纯装饰 SVG 移除 role，或改用 `<img alt>`。
2. **`color-contrast`（ch1 3 处）**
   - 焦点在 `#btnNextBottom` / 进度文字 / kbd 提示上对比度不足。优先调整 `css/slides.css` 与 `css/themes.css`。
3. **`landmark-one-main` / `region`（ch1）**
   - 文档结构缺少 `<main>` 包裹，progress bar 未被 landmark 包裹。
   - 建议：在 `slides.html` 模板里给 slide 容器套 `<main id="slideStage">`，并给 progress bar 配 `<aside>` 或 `<nav>`。
4. **`heading-order`（home 1）**
   - `#guide-intro-title` 跳级，与相邻 `<h1>/<h2>` 错位。修一处 DOM 即可。
5. **`aria-tooltip-name`（home 1）**
   - `.cm-tooltip` 缺少 `aria-label` / `aria-labelledby`。
6. **`nested-interactive`（home 1）**
   - 概念图 SVG 内部嵌套了可聚焦元素。建议外层不再赋予 `tabindex`，或把可聚焦子节点下沉到兄弟节点。

### 3.3 Best Practices（96）— 优先级：低

两项小分扣除通常为 CSP / 浏览器兼容性 / 第三方库告警。本次未在控制台抓取逐条 audit，**待复测时**结合
`--output html` 报告查具体失分项。

### 3.4 SEO（100）— 已达基线

无优化项。

## 4. 复测 / 复现方式

```bash
# 安装并启动 8091 端口静态服务（Lighthouse / axe 都依赖此端口）
python3 -m http.server 8091 &

# 单页 Lighthouse
CHROME_PATH=/usr/bin/google-chrome npx lighthouse http://127.0.0.1:8091/ \
  --output json --output-path docs/audits/lighthouse-home.json \
  --chrome-flags="--headless --no-sandbox --disable-dev-shm-usage" \
  --only-categories=performance,accessibility,best-practices,seo

# 单页 axe（注意 chromedriver 必须与本机 Chrome 严格同版本）
CHROME_TEST_PATH=/usr/bin/google-chrome \
CHROMEDRIVER_TEST_PATH=/path/to/chromedriver-149 \
npx @axe-core/cli http://127.0.0.1:8091/ --save docs/audits/axe-home.json \
  --chrome-options="--no-sandbox --disable-dev-shm-usage"

# 一键跑完首页 + CH1 slide 1（推荐）
bash scripts/run-audit.sh
# 等价于：
npm run audit:all
```

> 抓完后务必 `pkill -f "http.server 8091"`，避免遗留后台进程。
