# Hello-Agents 交互型学习 PPT 设计文档

> 项目：将 Datawhale《从零开始构建智能体》教程转化为交互式 HTML 演示
> 日期：2026-06-15
> 版本：v1.0

---

## 1. 项目概述

### 1.1 目标
将 GitHub 项目 [datawhalechina/hello-agents](https://github.com/datawhalechina/hello-agents) 的核心章节内容转化为一个**浏览器端的交互式 HTML 演示项目**，兼顾课堂教学与个人自学场景。

### 1.2 核心价值
- **可视化理解**：通过动画直观展示智能体工作流程（ReAct 循环、记忆流、协议交互等）
- **互动学习**：通过测验巩固知识，实时反馈学习效果
- **零依赖部署**：纯前端实现，单个 HTML 文件即可运行，便于分享和离线使用

### 1.3 目标用户
- AI 开发者、软件工程师、在校学生
- 有一定 Python 基础，希望系统学习智能体技术的自学者
- 需要教学工具的技术讲师

## 2. 内容规划

### 2.1 核心章节（6章）

| 章节 | 标题 | 核心知识点 | 交互重点 |
|------|------|-----------|----------|
| CH1 | 初识智能体 | 智能体定义、类型、架构 | 智能体类型对比图、架构组件可点击展开 |
| CH4 | 智能体经典范式构建 | ReAct、Plan-and-Solve、Reflection | **ReAct 循环动画**（Thought→Action→Observation）、范式对比 |
| CH7 | 构建你的Agent框架 | 从零构建框架、组件设计 | 框架组件关系图、代码结构可视化 |
| CH8 | 记忆与检索 | Memory系统、RAG | 记忆流动画（短期→长期→检索）、RAG流程图 |
| CH10 | 智能体通信协议 | MCP、A2A、ANP | 协议时序图、消息交互动画 |
| CH13 | 智能旅行助手 | 多智能体协作实战 | 多智能体协作流程图、角色分工可视化 |

### 2.2 每章结构

每章包含以下幻灯片类型：

1. **章节封面** - 标题、简介、学习目标
2. **概念讲解** - 核心概念的文字+图示说明
3. **交互演示** - 可视化动画（重点章节）
4. **代码展示** - 关键代码片段（语法高亮）
5. **知识测验** - 5-8道选择题/判断题
6. **章节小结** - 要点回顾、下章预告

## 3. 交互设计

### 3.1 概念可视化交互

#### 3.1.1 ReAct 循环动画（CH4）
- **触发方式**：点击"开始演示"按钮或按 Space 键
- **动画流程**：
  1. 用户问题输入框高亮
  2. Thought 气泡从 LLM 大脑中浮现（显示思考内容）
  3. Action 箭头指向工具箱，工具高亮
  4. Observation 结果返回，更新到记忆区
  5. 循环直到输出最终答案
- **控制按钮**：播放/暂停、单步前进、重置、速度调节（1x/2x）
- **状态显示**：当前步骤高亮，历史步骤变灰

#### 3.1.2 智能体架构图（CH1）
- **布局**：中心为 LLM 大脑，周围环绕组件（感知、记忆、工具、规划、执行）
- **交互**：点击组件展开详细说明面板，显示该组件的作用和示例
- **连线动画**：数据流向的动态连线

#### 3.1.3 记忆流可视化（CH8）
- **展示**：短期记忆（STM）→ 长期记忆（LTM）→ 检索（Retrieval）的流转
- **交互**：点击记忆条目查看详情，模拟记忆存储和检索过程

#### 3.1.4 协议时序图（CH10）
- **展示**：MCP 协议中 Host ↔ Client ↔ Server 的消息交互
- **动画**：消息按时间顺序依次发送，可暂停查看每条消息内容

#### 3.1.5 多智能体协作图（CH13）
- **展示**：旅行助手中 Planner、Booking、Advisor 等角色的协作
- **交互**：点击角色查看职责，观看任务分配和执行流程

### 3.2 知识检测交互

#### 3.2.1 测验系统
- **题型**：单选题、多选题、判断题
- **流程**：
  1. 显示题目和选项
  2. 用户选择后点击"提交"
  3. 实时显示正确/错误，绿色（正确）/红色（错误）高亮
  4. 显示解析说明
  5. 点击"下一题"继续
- **计分**：每题 10 分，章节满分 100 分
- **通过标准**：60 分及格，80 分良好，100 分优秀

#### 3.2.2 学习进度
- **全局进度条**：显示整体学习进度（6章）
- **章节进度**：显示当前章节的幻灯片进度和测验完成情况
- **完成标记**：已完成的章节显示勾选标记

## 4. 技术架构


### 4.2 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 结构 | HTML5 | 语义化标签 |
| 样式 | CSS3 | Flexbox/Grid 布局、CSS Variables、Animations |
| 逻辑 | Vanilla JavaScript (ES6+) | 零依赖，原生实现 |
| 动画 | CSS Animations + Canvas API | 简单动画用 CSS，复杂流程图用 Canvas |
| 代码高亮 | Prism.js (CDN) | 代码块语法高亮 |
| 存储 | LocalStorage | 学习进度、测验成绩本地存储 |
| 图标 | SVG Sprite | 内嵌 SVG，无外部字体依赖 |

### 4.3 关键设计决策

**决策 1：单页应用 vs 多页应用**
- 选择：单页应用（slides.html 动态加载章节内容）
- 理由：切换流畅，状态保持简单，适合演示场景

**决策 2：Canvas vs SVG 动画**
- 选择：简单动画用 CSS/SVG，复杂流程图用 Canvas
- 理由：CSS/SVG 易于维护，Canvas 适合高性能复杂动画

**决策 3：数据存储格式**
- 选择：JSON 配置文件 + LocalStorage 用户数据
- 理由：无需后端，纯静态部署，GitHub Pages 可直接托管

## 6. 动画设计规范

### 6.1 ReAct 循环动画（CH4）详细设计

**视觉元素**：
- 中心：LLM 大脑图标（圆形，紫色渐变）
- 下方：工具箱图标（方形，蓝色）
- 上方：用户问题输入框
- 右侧：记忆/历史记录区

**动画时序**：
```
时间轴（单步约 2 秒）：
T0: 用户问题高亮
T1: Thought 气泡从大脑浮现（文字逐字显示）
T2: Action 箭头从大脑指向工具箱（流动动画）
T3: 工具箱高亮，显示执行的工具名称
T4: Observation 箭头从工具箱返回大脑
T5: 观察结果文字显示
T6: 结果追加到记忆区
T7: 判断是否需要继续循环
```

**控制面板**：
- 播放/暂停按钮
- 单步前进按钮
- 重置按钮
- 速度滑块（0.5x - 2x）
- 当前步骤指示器（Step 1/5）

### 6.2 其他动画简述

| 动画 | 类型 | 核心效果 |
|------|------|----------|
| 智能体架构图 | SVG + CSS | 组件点击展开，连线流动 |
| 记忆流 | Canvas | 粒子效果表示记忆流转 |
| 协议时序图 | Canvas | 消息箭头按序发送，可点击查看详情 |
| 多智能体协作 | SVG + CSS | 角色图标间任务传递动画 |

## 8. 样式设计

### 8.1 色彩方案

- 主色：#6366F1（靛蓝）- 代表智能、科技

## 9. 性能考虑

1. **懒加载**：章节内容按需加载，非当前章节不渲染
2. **动画优化**：使用 requestAnimationFrame，避免重排重绘
3. **资源压缩**：图片使用 WebP 格式，代码压缩
4. **缓存策略**：LocalStorage 存储进度，减少重复计算
